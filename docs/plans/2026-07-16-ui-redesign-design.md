# AgenticOps 运维控制台 UI 重构设计

## 目标

将现有偏展示型、渐变和玻璃拟态较重的界面，重构为适合日常值守的专业运维控制台。视觉服务于事件识别、审批判断和异常处置，不改变现有业务流程、接口和权限模型。

## 设计原则

- 信息密度优先：在 1440px 桌面视口中优先展示更多有效信息，减少巨型卡片和装饰留白。
- 单一视觉语言：使用中性灰白工作区、深色导航和单一蓝色强调色；成功、警告、故障色只表达状态。
- 扁平层级：用边框、排版和间距建立层级，取消全局渐变、玻璃模糊与普遍阴影。
- 稳定交互：不使用位移、缩放、脉冲等装饰动画，仅保留 150–200ms 的颜色和透明度反馈。
- 可访问性：交互目标至少 44px，提供清晰焦点环，保证正文对比度，并尊重 reduced-motion。
- 渐进改造：保持 Vue 3、Tailwind 和 Lucide 技术栈，优先调整共享设计令牌与组件，不重写业务页面。

## 视觉系统

### 色彩

- 工作区背景：`#f4f6f8`
- 主表面：`#ffffff`
- 主文字：`#172033`
- 次级文字：`#526077`
- 边框：`#dfe3e8`
- 品牌强调：`#2457d6`
- 导航背景：`#111827`
- 成功、警告、故障分别使用低饱和绿色、琥珀色和红色，不参与品牌装饰。

### 尺寸与层级

- 常规圆角 6px，面板圆角 8px，弹窗圆角 10px。
- 常规面板不使用阴影；阴影只用于弹窗和浮层。
- 侧边栏桌面宽度 232px；工作区最大宽度 1680px。
- 页面标题 24px，区块标题 16px，正文 14px，辅助信息 12–13px。
- 数字指标启用 tabular numbers，避免数值变化时跳动。

## 组件方案

### 导航与布局

侧边栏保持现有信息架构和权限过滤，缩减宽度与间距，取消渐变、脉冲、发光和横向位移动效。当前路由使用纯色弱背景、左侧强调条和高对比文字。移动端将导航改为横向可滚动区域，避免纵向占满首屏。

### Dashboard

- 标题区从独立大卡片改为轻量页面头部，显示“运营驾驶舱”、说明和刷新时间。
- 四个 KPI 合并为一个连续指标带，通过分隔线而非四个浮动卡片组织。
- 数据源健康改为紧凑状态列表。
- 所有后端枚举在界面层映射为中文，保留原值只用于逻辑判断。
- 根因候选、热点问题簇和 Case 使用紧凑行式列表；状态、数量和跳转目标保持不变。
- 柱状比例使用单色强调条，不使用渐变。

### 表单、按钮与状态

- 主按钮使用纯色；次按钮使用白底边框；幽灵按钮只使用浅蓝底。
- 输入框和选择器统一 8px 圆角、44px 高度和清晰焦点环。
- Badge 从胶囊形收敛为 5px 圆角，避免所有状态都呈现为圆点药丸。
- 空状态高度降低，适配数据密集页面。

## 响应式与验证

- 检查 375px、768px、1024px、1440px 四个视口。
- 980px 以下布局切换为顶部导航；指标带与双列面板降为单列。
- 运行 TypeScript/Vite 构建和现有测试。
- 使用真实页面截图检查层级、溢出、文本截断、焦点态和空状态。

## 非目标

- 不更改接口、数据模型、权限、路由和审批流程。
- 不引入新的 UI 框架或动画依赖。
- 不使用营销页式图片、纹理、视差和不对称布局。

## 工具说明

本设计采用 `redesign-existing-projects` 的渐进审查原则和 `ui-ux-pro-max` 的可访问性规则。后者自带的设计系统检索脚本当前存在 Python f-string 语法错误，因此本次依据其静态规则与项目代码完成设计，不依赖脚本输出。
